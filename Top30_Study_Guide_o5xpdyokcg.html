253. Meeting Rooms II
Medium Intervals, Heap, Sorting
▶
📋 Problem Statement
Given an array of meeting intervals consisting of start and end times, determine the minimum number of conference rooms required to accommodate all meetings.
🎯 Why This Problem for This Role
This is THE capacity planning problem. 'Minimum rooms' maps directly to 'minimum servers/resources needed to serve all concurrent workloads.' Apple's iCloud team literally solves variants of this — how many instances do we need to handle overlapping request bursts?
⚠️ Edge Cases
  • Empty intervals array → return 0
  • Single meeting → return 1
  • All meetings overlap → return len(intervals)
  • No meetings overlap → return 1
  • Meetings that share exact start/end boundaries: [0,5], [5,10] → don't overlap
🔴 Brute Force
For every time point, count how many meetings are active. This is extremely slow for large time ranges.
def minMeetingRooms(intervals):
    if not intervals:
        return 0

    # Try every point in time, count overlapping meetings
    # Collect all unique time points
    times = set()
    for start, end in intervals:
        for t in range(start, end):
            times.add(t)

    max_rooms = 0
    for t in times:
        count = 0
        for start, end in intervals:
            if start <= t < end:
                count += 1
        max_rooms = max(max_rooms, count)

    return max_rooms
Time: O(n × T) where T is the time range span
Space: O(T)
🟢 Optimal Solution
Sort meetings by start time. Use a min-heap to track the earliest ending meeting. If the current meeting starts after the earliest ending one, we can reuse that room (pop it). Otherwise, we need a new room (push). The heap size at any point = rooms in use.
def minMeetingRooms(intervals):
    if not intervals:
        return 0

    import heapq

    # Sort by start time
    intervals.sort(key=lambda x: x[0])

    # Min-heap tracks end times of ongoing meetings
    heap = []  # stores end times

    for start, end in intervals:
        # If earliest ending meeting ends before this one starts,
        # reuse that room
        if heap and heap[0] <= start:
            heapq.heappop(heap)
        heapq.heappush(heap, end)

    # Heap size = number of rooms needed
    return len(heap)
Time: O(n log n)
Space: O(n)
56. Merge Intervals
Medium Intervals, Sorting
▶
📋 Problem Statement
Given an array of intervals where intervals[i] = [starti, endi], merge all overlapping intervals, and return an array of the non-overlapping intervals that cover all the intervals in the input.
🎯 Why This Problem for This Role
Merging overlapping time windows is fundamental to capacity planning. When you have overlapping demand periods, you merge them to find the true contiguous high-demand windows. Also directly relevant to consolidating monitoring alerts.
⚠️ Edge Cases
  • Single interval → return as-is
  • No overlapping intervals → return sorted list
  • All intervals overlap into one → return single merged interval
  • Intervals that touch at endpoints: [1,4],[4,5] → merge to [1,5]
  • Intervals already sorted vs. unsorted input
🔴 Brute Force
Repeatedly scan all pairs, merge overlapping ones, until no more merges happen. Very inefficient due to repeated scanning.
def merge(intervals):
    if not intervals:
        return []

    # Repeatedly scan and merge overlapping pairs
    merged = [list(i) for i in intervals]
    changed = True

    while changed:
        changed = False
        new_merged = []
        used = [False] * len(merged)

        for i in range(len(merged)):
            if used[i]:
                continue
            curr = list(merged[i])
            for j in range(i + 1, len(merged)):
                if used[j]:
                    continue
                # Check overlap
                if curr[0] <= merged[j][1] and merged[j][0] <= curr[1]:
                    curr[0] = min(curr[0], merged[j][0])
                    curr[1] = max(curr[1], merged[j][1])
                    used[j] = True
                    changed = True
            new_merged.append(curr)
        merged = new_merged

    return sorted(merged)
Time: O(n³) worst case with repeated passes
Space: O(n)
🟢 Optimal Solution
Sort by start time. Then iterate: if the current interval overlaps with the last merged interval (start ≤ previous end), extend the end. Otherwise, start a new merged interval. One pass after sorting.
def merge(intervals):
    if not intervals:
        return []

    # Sort by start time
    intervals.sort(key=lambda x: x[0])

    merged = [intervals[0]]

    for start, end in intervals[1:]:
        # If current interval overlaps with last merged
        if start <= merged[-1][1]:
            merged[-1][1] = max(merged[-1][1], end)
        else:
            merged.append([start, end])

    return merged
Time: O(n log n)
Space: O(n) for output
252. Meeting Rooms
Easy Intervals, Sorting
▶
📋 Problem Statement
Given an array of meeting time intervals where intervals[i] = [start, end], determine if a person could attend all meetings (i.e., no two meetings overlap).
🎯 Why This Problem for This Role
The simpler version of Meeting Rooms II. Tests whether you can detect scheduling conflicts — a basic check before doing capacity allocation. In infrastructure terms: can one server handle all these tasks sequentially?
⚠️ Edge Cases
  • Empty array → return True (no conflicts)
  • Single meeting → return True
  • Two meetings sharing an endpoint: [0,5],[5,10] → no conflict (True)
  • Two meetings sharing an endpoint: [0,5],[4,10] → conflict (False)
  • All identical meetings → conflict
🔴 Brute Force
Compare every pair of meetings to check for overlap. Simple but slow for large inputs.
def canAttendMeetings(intervals):
    # Check every pair for overlap
    for i in range(len(intervals)):
        for j in range(i + 1, len(intervals)):
            # Overlap if one starts before the other ends
            if intervals[i][0] < intervals[j][1] and intervals[j][0] < intervals[i][1]:
                return False
    return True
Time: O(n²)
Space: O(1)
🟢 Optimal Solution
Sort by start time. Then just check each consecutive pair — if any meeting starts before the previous one ends, there's a conflict. After sorting, we only need to check neighbors.
def canAttendMeetings(intervals):
    # Sort by start time
    intervals.sort(key=lambda x: x[0])

    # Check adjacent pairs for overlap
    for i in range(1, len(intervals)):
        if intervals[i][0] < intervals[i - 1][1]:
            return False

    return True
Time: O(n log n)
Space: O(1) if in-place sort
134. Gas Station
Medium Greedy, Array
▶
📋 Problem Statement
There are n gas stations along a circular route. Given arrays gas and cost where gas[i] is the gas gained at station i and cost[i] is the gas needed to reach the next station, find the starting station index to complete the circuit, or return -1.
🎯 Why This Problem for This Role
This is a resource-sufficiency problem: 'Given a workload cycle and resource availability at each point, where should we start provisioning to ensure we never run out?' Directly maps to capacity planning where you need to find the optimal starting configuration.
⚠️ Edge Cases
  • Total gas < total cost → impossible, return -1
  • Total gas == total cost → exactly one valid start
  • Single station with enough gas → return 0
  • All stations identical → return 0
  • Start must be at the last station
🔴 Brute Force
Try each station as a starting point. Simulate the full circuit. If tank never goes negative, that's a valid start.
def canCompleteCircuit(gas, cost):
    n = len(gas)

    for start in range(n):
        tank = 0
        can_complete = True

        for i in range(n):
            station = (start + i) % n
            tank += gas[station] - cost[station]
            if tank < 0:
                can_complete = False
                break

        if can_complete:
            return start

    return -1
Time: O(n²)
Space: O(1)
🟢 Optimal Solution
Key insight: if total gas ≥ total cost, a solution exists. As we iterate, if the running tank goes negative at station i, no station from our current start through i can be the answer (they'd all fail at or before i). So we reset start to i+1 and reset tank. The last start standing is the answer.
def canCompleteCircuit(gas, cost):
    # If total gas < total cost, impossible
    if sum(gas) < sum(cost):
        return -1

    tank = 0
    start = 0

    for i in range(len(gas)):
        tank += gas[i] - cost[i]
        # If tank goes negative, we can't start from 'start'
        # or any station between start and i
        if tank < 0:
            start = i + 1
            tank = 0

    return start
Time: O(n)
Space: O(1)
347. Top K Frequent Elements
Medium Hash Table, Heap, Sorting
▶
📋 Problem Statement
Given an integer array nums and an integer k, return the k most frequent elements. You may return the answer in any order.
🎯 Why This Problem for This Role
In capacity planning, you constantly need 'top K' analysis: top K busiest services, top K resource consumers, top K error codes. This pattern underpins every monitoring dashboard at Apple.
⚠️ Edge Cases
  • k equals number of unique elements → return all unique elements
  • k = 1 → return the single most frequent
  • All elements same → return that element
  • All elements appear once → return any k of them
  • Array with exactly k distinct elements
🔴 Brute Force
Count frequencies with a hash map, then sort all unique elements by frequency. Return the top k. Sorting makes it O(n log n).
def topKFrequent(nums, k):
    from collections import Counter

    # Count frequencies, sort by frequency
    count = Counter(nums)
    # Sort all items by frequency descending
    sorted_items = sorted(count.keys(), key=lambda x: count[x], reverse=True)

    return sorted_items[:k]
Time: O(n log n)
Space: O(n)
🟢 Optimal Solution
Use bucket sort where index = frequency. Since max frequency ≤ n, we create n+1 buckets. Place each element in its frequency bucket, then collect from the highest bucket down until we have k elements. Avoids the O(n log n) sort.
def topKFrequent(nums, k):
    from collections import Counter

    count = Counter(nums)

    # Bucket sort: index = frequency, value = list of elements
    # Max possible frequency is len(nums)
    buckets = [[] for _ in range(len(nums) + 1)]

    for num, freq in count.items():
        buckets[freq].append(num)

    # Collect from highest frequency bucket down
    result = []
    for freq in range(len(buckets) - 1, 0, -1):
        for num in buckets[freq]:
            result.append(num)
            if len(result) == k:
                return result

    return result
Time: O(n)
Space: O(n)
362. Design Hit Counter
Medium Design, Queue
▶
📋 Problem Statement
Design a hit counter which counts the number of hits received in the past 5 minutes (300 seconds). Implement hit(timestamp) and getHits(timestamp).
🎯 Why This Problem for This Role
This IS service observability. Apple's iCloud team builds exactly this — real-time request-rate counters for monitoring QPS, error rates, and traffic bursts. The rolling time window is a core pattern in their monitoring infrastructure.
⚠️ Edge Cases
  • Multiple hits at the same timestamp
  • getHits called with no prior hits → 0
  • Hits exactly 300 seconds apart (boundary check)
  • Timestamps arriving in strict order (guaranteed by problem)
  • Very sparse hits (large gaps between timestamps)
🔴 Brute Force
Store every hit timestamp. On getHits, scan all timestamps and count those within the 300-second window. Memory grows unbounded.
class HitCounter:
    def __init__(self):
        self.hits = []  # store all timestamps

    def hit(self, timestamp):
        self.hits.append(timestamp)

    def getHits(self, timestamp):
        # Count hits in [timestamp - 299, timestamp]
        count = 0
        for t in self.hits:
            if t > timestamp - 300 and t <= timestamp:
                count += 1
        return count
Time: hit: O(1), getHits: O(n) where n = total hits
Space: O(n) — stores all hits forever
🟢 Optimal Solution
Use two fixed arrays of size 300 (the window size). Each slot maps to a second via modulo. On hit, check if the slot's timestamp matches; if not, reset the count. On getHits, sum counts for slots whose timestamp is within the 300-second window. Constant space regardless of hit volume.
class HitCounter:
    def __init__(self):
        # Fixed-size arrays of 300 (one per second in window)
        self.times = [0] * 300
        self.counts = [0] * 300

    def hit(self, timestamp):
        idx = timestamp % 300
        if self.times[idx] != timestamp:
            # New second, reset count
            self.times[idx] = timestamp
            self.counts[idx] = 1
        else:
            self.counts[idx] += 1

    def getHits(self, timestamp):
        total = 0
        for i in range(300):
            # Only count if the stored time is within the window
            if timestamp - self.times[i] < 300:
                total += self.counts[i]
        return total
Time: hit: O(1), getHits: O(300) = O(1)
Space: O(300) = O(1)
981. Time Based Key-Value Store
Medium Design, Binary Search, Hash Table
▶
📋 Problem Statement
Design a time-based key-value data structure that supports setting values at a specified time and retrieving the value at a certain timestamp. Implement set(key, value, timestamp) and get(key, timestamp) which returns the value with the largest timestamp ≤ given timestamp.
🎯 Why This Problem for This Role
Time-series data retrieval is the backbone of ML forecasting pipelines. This data structure models exactly how capacity metrics are stored and queried — 'what was the CPU utilization at time T?' Apple's monitoring systems use variants of this pattern.
⚠️ Edge Cases
  • get() with timestamp before any set → return ''
  • get() with exact timestamp match
  • get() between two timestamps → return earlier one
  • Multiple values for same key at different timestamps
  • Single key with single timestamp
🔴 Brute Force
Store all (timestamp, value) pairs per key. On get, linearly scan to find the latest timestamp ≤ target. Slow for keys with many entries.
class TimeMap:
    def __init__(self):
        self.store = {}  # key -> [(timestamp, value), ...]

    def set(self, key, value, timestamp):
        if key not in self.store:
            self.store[key] = []
        self.store[key].append((timestamp, value))

    def get(self, key, timestamp):
        if key not in self.store:
            return ""

        # Linear scan for largest timestamp <= given
        result = ""
        for ts, val in self.store[key]:
            if ts <= timestamp:
                result = val
        return result
Time: set: O(1), get: O(n)
Space: O(n)
🟢 Optimal Solution
Since timestamps are given in increasing order per the problem constraints, the list per key is already sorted. Use binary search to find the largest timestamp ≤ target. This is the standard approach for time-series lookups.
class TimeMap:
    def __init__(self):
        self.store = {}  # key -> [(timestamp, value), ...]

    def set(self, key, value, timestamp):
        if key not in self.store:
            self.store[key] = []
        self.store[key].append((timestamp, value))

    def get(self, key, timestamp):
        if key not in self.store:
            return ""

        values = self.store[key]
        # Binary search for largest timestamp <= given
        lo, hi = 0, len(values) - 1
        result = ""

        while lo <= hi:
            mid = (lo + hi) // 2
            if values[mid][0] <= timestamp:
                result = values[mid][1]
                lo = mid + 1
            else:
                hi = mid - 1

        return result
Time: set: O(1), get: O(log n)
Space: O(n)
1146. Snapshot Array
Medium Design, Binary Search
▶
📋 Problem Statement
Implement a SnapshotArray supporting set(index, val), snap() which takes a snapshot and returns snap_id, and get(index, snap_id) which returns the value at index at the given snapshot.
🎯 Why This Problem for This Role
Point-in-time snapshots are essential for capacity planning. 'What was our resource allocation at snapshot T?' This models versioned configuration and metric stores that Apple uses for tracking infrastructure state over time.
⚠️ Edge Cases
  • get() on an index that was never set → return 0
  • Multiple snaps with no changes in between
  • set() the same index multiple times before a snap
  • get() with snap_id 0 (first snapshot)
  • Large array length but sparse updates
🔴 Brute Force
Copy the entire array on each snap(). get() is O(1) but memory is massive — each snapshot stores the full array even if only one element changed.
class SnapshotArray:
    def __init__(self, length):
        self.arr = [0] * length
        self.snaps = []  # list of full array copies

    def set(self, index, val):
        self.arr[index] = val

    def snap(self):
        self.snaps.append(self.arr[:])  # full copy
        return len(self.snaps) - 1

    def get(self, index, snap_id):
        return self.snaps[snap_id][index]
Time: set: O(1), snap: O(n), get: O(1)
Space: O(n × S) where S = number of snapshots
🟢 Optimal Solution
Instead of copying the whole array, store only the changes per index as (snap_id, value) pairs. snap() just increments the ID. get() binary-searches the history for the latest entry at or before the target snap_id. Only stores what actually changed.
class SnapshotArray:
    def __init__(self, length):
        # Each index stores a list of (snap_id, value) pairs
        self.changes = [[[0, 0]] for _ in range(length)]
        self.snap_id = 0

    def set(self, index, val):
        self.changes[index].append([self.snap_id, val])

    def snap(self):
        self.snap_id += 1
        return self.snap_id - 1

    def get(self, index, snap_id):
        # Binary search for the latest entry with snap_id <= target
        history = self.changes[index]
        lo, hi = 0, len(history) - 1

        while lo <= hi:
            mid = (lo + hi) // 2
            if history[mid][0] <= snap_id:
                lo = mid + 1
            else:
                hi = mid - 1

        return history[hi][1]
Time: set: O(1), snap: O(1), get: O(log S)
Space: O(number of set calls)

146. LRU Cache
Medium Design, Hash Table, Linked List
▶
📋 Problem Statement
Design a data structure that follows LRU eviction with O(1) get(key) and put(key, value). If capacity is exceeded, evict the least recently used key.
🎯 Why This Problem for This Role
LRU caching is fundamental to cloud infrastructure. Apple's CDN, object storage, and service layers all use LRU or LRU-variant caches. This problem tests whether you can combine hash maps and linked lists for O(1) operations — a key systems engineering skill.
⚠️ Edge Cases
  • put() on an existing key (update value and move to most recent)
  • get() on a non-existent key → return -1
  • Capacity of 1 (every put evicts the previous)
  • put() then immediate get() on same key
  • Eviction order when multiple keys are equally old
🔴 Brute Force
Use a list to track usage order. On access, remove from current position and append to end. On eviction, pop from front. The list operations are O(n).
class LRUCache:
    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = {}        # key -> value
        self.order = []        # list of keys in usage order

    def get(self, key):
        if key not in self.cache:
            return -1
        # Move to end (most recently used)
        self.order.remove(key)  # O(n)
        self.order.append(key)
        return self.cache[key]

    def put(self, key, value):
        if key in self.cache:
            self.order.remove(key)  # O(n)
        elif len(self.cache) >= self.capacity:
            # Evict least recently used
            lru = self.order.pop(0)  # O(n)
            del self.cache[lru]

        self.cache[key] = value
        self.order.append(key)
Time: O(n) for both get and put due to list.remove()
Space: O(capacity)
🟢 Optimal Solution
Combine a hash map (key→node) with a doubly-linked list. The list maintains usage order: most recent at tail, least recent at head. Remove and insert are O(1) with direct node references. Dummy head/tail nodes simplify edge cases.
class Node:
    def __init__(self, key=0, val=0):
        self.key = key
        self.val = val
        self.prev = None
        self.next = None

class LRUCache:
    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = {}  # key -> Node
        # Dummy head and tail for easy manipulation
        self.head = Node()
        self.tail = Node()
        self.head.next = self.tail
        self.tail.prev = self.head

    def _remove(self, node):
        node.prev.next = node.next
        node.next.prev = node.prev

    def _add_to_end(self, node):
        node.prev = self.tail.prev
        node.next = self.tail
        self.tail.prev.next = node
        self.tail.prev = node

    def get(self, key):
        if key not in self.cache:
            return -1
        node = self.cache[key]
        self._remove(node)
        self._add_to_end(node)
        return node.val

    def put(self, key, value):
        if key in self.cache:
            self._remove(self.cache[key])
            del self.cache[key]

        node = Node(key, value)
        self._add_to_end(node)
        self.cache[key] = node

        if len(self.cache) > self.capacity:
            # Evict LRU (node right after dummy head)
            lru = self.head.next
            self._remove(lru)
            del self.cache[lru.key]
Time: O(1) for both get and put
Space: O(capacity)
706. Design HashMap
Medium Design, Hash Table
▶
📋 Problem Statement
Design a HashMap without using any built-in hash table libraries. Implement put(key, value), get(key), and remove(key).
🎯 Why This Problem for This Role
Understanding hash map internals is core systems knowledge. At Apple, you'll work with custom data structures for performance-critical paths. This tests whether you understand hashing, collision resolution, and load factor tradeoffs.
⚠️ Edge Cases
  • put() then get() on same key
  • put() with existing key (update)
  • remove() on non-existent key (no-op)
  • get() on non-existent key → -1
  • Hash collisions (multiple keys mapping to same bucket)
🔴 Brute Force
Direct addressing with a massive array. Works within the problem constraints but wastes enormous memory. Not a real hash map.
class MyHashMap:
    def __init__(self):
        # Simple: use a large array (wasteful but works)
        self.data = [-1] * 1000001

    def put(self, key, value):
        self.data[key] = value

    def get(self, key):
        return self.data[key]

    def remove(self, key):
        self.data[key] = -1
Time: O(1) for all operations
Space: O(max_key) — 10^6 array regardless of usage
🟢 Optimal Solution
Use an array of buckets with chaining (each bucket is a list of (key, value) pairs). Hash function = key % size. On collision, search the chain linearly. With a good bucket count, chains stay short and operations are effectively O(1).
class MyHashMap:
    def __init__(self):
        self.size = 1000
        self.buckets = [[] for _ in range(self.size)]

    def _hash(self, key):
        return key % self.size

    def put(self, key, value):
        idx = self._hash(key)
        for i, (k, v) in enumerate(self.buckets[idx]):
            if k == key:
                self.buckets[idx][i] = (key, value)
                return
        self.buckets[idx].append((key, value))

    def get(self, key):
        idx = self._hash(key)
        for k, v in self.buckets[idx]:
            if k == key:
                return v
        return -1

    def remove(self, key):
        idx = self._hash(key)
        for i, (k, v) in enumerate(self.buckets[idx]):
            if k == key:
                self.buckets[idx].pop(i)
                return
Time: Average O(1), worst O(n/k) per operation where k = bucket count
Space: O(k + n) where n = stored elements
155. Min Stack
Medium Design, Stack
▶
📋 Problem Statement
Design a stack supporting push, pop, top, and getMin — all in O(1) time.
🎯 Why This Problem for This Role
Maintaining auxiliary metadata (like min/max) alongside a data structure is a pattern used everywhere in monitoring systems. Think: tracking the minimum latency seen so far as you process a stream of requests.
⚠️ Edge Cases
  • Push minimum, then pop it → new min must be correct
  • All elements are the same
  • Push in decreasing order (min changes every push)
  • Push in increasing order (min never changes)
  • Single element: push then getMin then pop
🔴 Brute Force
Use a regular stack. getMin scans the entire stack each time. Simple but doesn't meet the O(1) requirement for getMin.
class MinStack:
    def __init__(self):
        self.stack = []

    def push(self, val):
        self.stack.append(val)

    def pop(self):
        self.stack.pop()

    def top(self):
        return self.stack[-1]

    def getMin(self):
        return min(self.stack)  # O(n) scan
Time: push/pop/top: O(1), getMin: O(n)
Space: O(n)
🟢 Optimal Solution
Maintain a parallel min_stack that only stores values when they become a new minimum. The top of min_stack is always the current minimum. When we pop a value equal to the current min, we pop from both stacks. Use <= (not <) to handle duplicate minimums.
class MinStack:
    def __init__(self):
        self.stack = []      # stores values
        self.min_stack = []  # stores running minimums

    def push(self, val):
        self.stack.append(val)
        # Push to min_stack if it's empty or val <= current min
        if not self.min_stack or val <= self.min_stack[-1]:
            self.min_stack.append(val)

    def pop(self):
        val = self.stack.pop()
        # If popped value is the current min, pop from min_stack too
        if val == self.min_stack[-1]:
            self.min_stack.pop()

    def top(self):
        return self.stack[-1]

    def getMin(self):
        return self.min_stack[-1]
Time: O(1) for all operations
Space: O(n)
622. Design Circular Queue
Medium Design, Queue
▶
📋 Problem Statement
Design a circular queue (ring buffer) supporting enQueue, deQueue, Front, Rear, isEmpty, and isFull operations.
🎯 Why This Problem for This Role
Ring buffers are used everywhere in infrastructure: log buffers, network packet queues, metric collection pipelines. Apple's iCloud systems use circular buffers for efficient, bounded-memory data streaming.
⚠️ Edge Cases
  • Enqueue when full → return False
  • Dequeue when empty → return False
  • Front/Rear when empty → return -1
  • Fill completely, empty completely, fill again (wrap-around)
  • Queue of size 1
🔴 Brute Force
Use a Python list as the underlying storage. deQueue uses pop(0) which shifts all elements — O(n). Not a true circular buffer.
class MyCircularQueue:
    def __init__(self, k):
        self.queue = []
        self.capacity = k

    def enQueue(self, value):
        if len(self.queue) >= self.capacity:
            return False
        self.queue.append(value)
        return True

    def deQueue(self):
        if not self.queue:
            return False
        self.queue.pop(0)  # O(n) shift
        return True

    def Front(self):
        return self.queue[0] if self.queue else -1

    def Rear(self):
        return self.queue[-1] if self.queue else -1

    def isEmpty(self):
        return len(self.queue) == 0

    def isFull(self):
        return len(self.queue) == self.capacity
Time: deQueue: O(n) due to list.pop(0), others: O(1)
Space: O(k)
🟢 Optimal Solution
Use a fixed-size array with head pointer and count. Tail is computed as (head + count) % capacity. On deQueue, advance head. No element shifting needed. The modulo arithmetic handles the wrap-around.
class MyCircularQueue:
    def __init__(self, k):
        self.queue = [0] * k
        self.capacity = k
        self.head = 0
        self.count = 0  # current number of elements

    def enQueue(self, value):
        if self.isFull():
            return False
        # Tail position
        tail = (self.head + self.count) % self.capacity
        self.queue[tail] = value
        self.count += 1
        return True

    def deQueue(self):
        if self.isEmpty():
            return False
        self.head = (self.head + 1) % self.capacity
        self.count -= 1
        return True

    def Front(self):
        if self.isEmpty():
            return -1
        return self.queue[self.head]

    def Rear(self):
        if self.isEmpty():
            return -1
        tail = (self.head + self.count - 1) % self.capacity
        return self.queue[tail]

    def isEmpty(self):
        return self.count == 0

    def isFull(self):
        return self.count == self.capacity
Time: O(1) for all operations
Space: O(k)

🟢 Optimal Solution
Combine a list (for O(1) random access via random.choice) with a hash map (value→index for O(1) lookup). The key trick for O(1) remove: swap the target element with the last element, update the map, then pop from the end. No shifting needed.
import random

class RandomizedSet:
    def __init__(self):
        self.val_to_idx = {}  # value -> index in list
        self.values = []       # list of values

    def insert(self, val):
        if val in self.val_to_idx:
            return False
        self.val_to_idx[val] = len(self.values)
        self.values.append(val)
        return True

    def remove(self, val):
        if val not in self.val_to_idx:
            return False
        # Swap with last element, then pop
        idx = self.val_to_idx[val]
        last = self.values[-1]
        self.values[idx] = last
        self.val_to_idx[last] = idx
        self.values.pop()
        del self.val_to_idx[val]
        return True

    def getRandom(self):
        return random.choice(self.values)
Time: O(1) average for all operations
Space: O(n)

Try every possible subarray by fixing start and extending end. Track the maximum sum seen. Correct but slow.
def maxSubArray(nums):

    for i in range(len(nums)):
        current_sum = 0
        for j in range(i, len(nums)):
            current_sum += nums[j]
            max_sum = max(max_sum, current_sum)

def maxSubArray(nums):
    max_sum = nums[0]
    current_sum = nums[0]

    for i in range(1, len(nums)):
        # Either extend the current subarray or start fresh
        current_sum = max(nums[i], current_sum + nums[i])
        max_sum = max(max_sum, current_sum)

The prefix/suffix decomposition pattern is widely used in time-series feature engineering. In ML pipelines for capacity planning, you often compute features like 'total load excluding this service' — same pattern.
⚠️ Edge Cases
  • Array contains a zero → only that position has non-zero product
  • Array contains two zeros → all products are zero
  • Array of all ones
  • Array with negative numbers
  • Array of length 2
🔴 Brute Force
For each index, multiply all other elements. Two nested loops.
def productExceptSelf(nums):
    n = len(nums)
    result = []

    for i in range(n):
        product = 1
        for j in range(n):
            if i != j:
                product *= nums[j]
        result.append(product)

    return result
Time: O(n²)
Space: O(n) for output
🟢 Optimal Solution
Two passes: first left-to-right building prefix products, then right-to-left multiplying by suffix products. Each element gets the product of everything to its left times everything to its right. No division needed.
def productExceptSelf(nums):
    n = len(nums)
    result = [1] * n

    # Left pass: result[i] = product of all elements to the left
    left_product = 1
    for i in range(n):
        result[i] = left_product
        left_product *= nums[i]

    # Right pass: multiply by product of all elements to the right
    right_product = 1
    for i in range(n - 1, -1, -1):
        result[i] *= right_product
        right_product *= nums[i]

    return result
Time: O(n)
Space: O(1) extra (output array doesn't count)

Medium Array, Hash Table
▶
📋 Problem Statement

🎯 Why This Problem for This Role
Finding consecutive ranges in data is relevant to identifying contiguous time periods of high load, consecutive alert windows, or sequential resource IDs that need attention.
⚠️ Edge Cases
  • Empty array → return 0
  • All duplicates: [1,1,1] → return 1
  • Single element → return 1
  • Already sorted consecutive → return length
  • Multiple sequences of same length
🔴 Brute Force
Sort the array, then scan for consecutive runs. Simple but violates the O(n) requirement.
def longestConsecutive(nums):
    if not nums:
        return 0

    nums.sort()
    max_len = 1
    curr_len = 1

    for i in range(1, len(nums)):
        if nums[i] == nums[i-1]:
            continue  # skip duplicates
        if nums[i] == nums[i-1] + 1:
            curr_len += 1
        else:
            curr_len = 1
        max_len = max(max_len, curr_len)

    return max_len
Time: O(n log n) due to sorting
Space: O(1) if in-place sort
🟢 Optimal Solution
Put all numbers in a set. For each number, only start counting if it's the start of a sequence (num-1 not in set). Then count forward. Each number is visited at most twice (once in the outer loop, once in a while loop), so total work is O(n).
def longestConsecutive(nums):
    num_set = set(nums)
    max_len = 0

    for num in num_set:
        # Only start counting from the beginning of a sequence
        if num - 1 not in num_set:
            current = num
            length = 1

            while current + 1 in num_set:
                current += 1
                length += 1

            max_len = max(max_len, length)

    return max_len
Time: O(n)
Space: O(n)

207. Course Schedule
Medium Graph, Topological Sort, DFS
▶
📋 Problem Statement
There are numCourses courses with prerequisites. Given prerequisites[i] = [a, b] meaning you must take b before a, return true if you can finish all courses (i.e., no cycles).
🎯 Why This Problem for This Role
This is cycle detection in a dependency graph. At Apple, workloads have dependencies — 'service A needs data from service B.' If there's a circular dependency, the system deadlocks. This problem directly tests that understanding.
⚠️ Edge Cases
  • No prerequisites → return True
  • Self-loop: [1,1] → return False
  • Two courses with mutual dependency → cycle → False
  • Linear chain of dependencies → True
  • Disconnected components (some courses independent)
🔴 Brute Force
DFS with three-color marking. This IS already optimal — the 'brute force' approach for cycle detection is DFS, which is efficient. The distinction here is DFS vs BFS (Kahn's) approach.
def canFinish(numCourses, prerequisites):
    from collections import defaultdict

    graph = defaultdict(list)
    for a, b in prerequisites:
        graph[b].append(a)

    # Try to find a valid ordering using DFS
    # States: 0=unvisited, 1=in-progress, 2=completed
    state = [0] * numCourses

    def has_cycle(node):
        if state[node] == 1:  # back edge = cycle
            return True
        if state[node] == 2:  # already processed
            return False

        state[node] = 1
        for neighbor in graph[node]:
            if has_cycle(neighbor):
                return True
        state[node] = 2
        return False

    for i in range(numCourses):
        if has_cycle(i):
            return False
    return True
Time: O(V + E)
Space: O(V + E)
🟢 Optimal Solution
Kahn's algorithm (BFS topological sort). Compute in-degrees. Start with all zero-in-degree nodes. Process each, decrement neighbors' in-degrees. If a neighbor reaches 0, add to queue. If we process all nodes, no cycle exists. More intuitive for scheduling contexts.
def canFinish(numCourses, prerequisites):
    from collections import defaultdict, deque

    graph = defaultdict(list)
    in_degree = [0] * numCourses

    for a, b in prerequisites:
        graph[b].append(a)
        in_degree[a] += 1

    # Kahn's algorithm: BFS starting from nodes with 0 in-degree
    queue = deque()
    for i in range(numCourses):
        if in_degree[i] == 0:
            queue.append(i)

    completed = 0
    while queue:
        node = queue.popleft()
        completed += 1

        for neighbor in graph[node]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)

    return completed == numCourses
Time: O(V + E)
Space: O(V + E)
210. Course Schedule II
Medium Graph, Topological Sort
▶
📋 Problem Statement
Given numCourses and prerequisites, return a valid ordering of courses. If impossible (cycle exists), return an empty array.
🎯 Why This Problem for This Role
This extends Course Schedule I to actually produce the ordering. In capacity planning, you need to determine the order to provision resources: 'set up the database before the app server, set up networking before the database.' This IS topological sort of a deployment pipeline.
⚠️ Edge Cases
  • No prerequisites → return any ordering [0,1,...,n-1]
  • Cycle exists → return []
  • Linear chain → only one valid ordering
  • Multiple valid orderings (any is acceptable)
  • Single course → return [0]
🔴 Brute Force
DFS-based topological sort. Append nodes in post-order (after all descendants are processed), then reverse. Detects cycles via back-edge detection (node in 'in-progress' state).
def findOrder(numCourses, prerequisites):
    from collections import defaultdict

    graph = defaultdict(list)
    for a, b in prerequisites:
        graph[b].append(a)

    state = [0] * numCourses  # 0=unvisited, 1=in-progress, 2=done
    order = []

    def dfs(node):
        if state[node] == 1:
            return False  # cycle
        if state[node] == 2:
            return True

        state[node] = 1
        for neighbor in graph[node]:
            if not dfs(neighbor):
                return False

        state[node] = 2
        order.append(node)  # post-order
        return True

    for i in range(numCourses):
        if not dfs(i):
            return []

    return order[::-1]  # reverse post-order = topological order
Time: O(V + E)
Space: O(V + E)
🟢 Optimal Solution
Kahn's algorithm again, but this time we collect the processing order. Nodes are added to the result as they're dequeued (all their prerequisites are satisfied). If result length < numCourses, a cycle exists.
def findOrder(numCourses, prerequisites):
    from collections import defaultdict, deque

    graph = defaultdict(list)
    in_degree = [0] * numCourses

    for a, b in prerequisites:
        graph[b].append(a)
        in_degree[a] += 1

    queue = deque()
    for i in range(numCourses):
        if in_degree[i] == 0:
            queue.append(i)

    order = []
    while queue:
        node = queue.popleft()
        order.append(node)

        for neighbor in graph[node]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)

    return order if len(order) == numCourses else []
Time: O(V + E)
Space: O(V + E)

def numIslands(grid):
    if not grid:
        return 0

    rows, cols = len(grid), len(grid[0])
    visited = [[False] * cols for _ in range(rows)]
    count = 0

    def dfs(r, c):
        if (r < 0 or r >= rows or c < 0 or c >= cols
                or grid[r][c] == '0' or visited[r][c]):
            return
        visited[r][c] = True
        dfs(r+1, c)
        dfs(r-1, c)
        dfs(r, c+1)
        dfs(r, c-1)

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == '1' and not visited[r][c]:

def numIslands(grid):

        return 0

        dfs(r-1, c)
        dfs(r, c+1)
        dfs(r, c-1)

        for c in range(cols):
            if grid[r][c] == '1':
                dfs(r, c)
                count += 1

Linear scan to find first and last occurrence. Correct but doesn't leverage the sorted property.
def searchRange(nums, target):

    last = -1

    for i in range(len(nums)):
        if nums[i] == target:
            if first == -1:
                first = i
            last = i

    return [first, last]
Time: O(n)
Space: O(1)
🟢 Optimal Solution
Two binary searches: one biased left (when found, continue searching left half) and one biased right (when found, continue searching right half). Each finds one boundary in O(log n).
def searchRange(nums, target):
    def find_left(nums, target):
        lo, hi = 0, len(nums) - 1
        result = -1
        while lo <= hi:
            mid = (lo + hi) // 2
            if nums[mid] == target:
                result = mid
                hi = mid - 1  # keep searching left
            elif nums[mid] < target:
                lo = mid + 1
            else:
                hi = mid - 1
        return result

    def find_right(nums, target):
        lo, hi = 0, len(nums) - 1
        result = -1
        while lo <= hi:
            mid = (lo + hi) // 2
            if nums[mid] == target:
                result = mid
                lo = mid + 1  # keep searching right
            elif nums[mid] < target:
                lo = mid + 1
            else:
                hi = mid - 1
        return result

    return [find_left(nums, target), find_right(nums, target)]
Time: O(log n)
Space: O(1)

        self.val = val
        self.left = left
        self.right = right

def sortedArrayToBST(nums):
    if not nums:
        return None

    def build(left, right):
        if left > right:
            return None

        mid = (left + right) // 2
        node = TreeNode(nums[mid])
        node.left = build(left, mid - 1)
        node.right = build(mid + 1, right)
        return node

    return build(0, len(nums) - 1)

3. Longest Substring Without Repeating Characters
Medium Sliding Window, Hash Table, String
▶
📋 Problem Statement
Given a string s, find the length of the longest substring without repeating characters.
🎯 Why This Problem for This Role
The sliding window pattern is essential for time-window analysis in monitoring. Think: 'what's the longest period where all monitored metrics were unique/within bounds?' This is the canonical sliding window problem.
⚠️ Edge Cases
  • Empty string → 0
  • All same characters: 'aaaa' → 1
  • All unique characters → return full length
  • Repeating character at the start and end
  • String of length 1 → 1
🔴 Brute Force
For each starting position, extend until a duplicate is found. Track the longest valid window. Redundant work: doesn't reuse information from previous windows.
def lengthOfLongestSubstring(s):
    max_len = 0

    for i in range(len(s)):
        seen = set()
        for j in range(i, len(s)):
            if s[j] in seen:
                break
            seen.add(s[j])
            max_len = max(max_len, j - i + 1)

    return max_len
Time: O(n²)
Space: O(min(n, charset))
🟢 Optimal Solution
Sliding window with a hash map tracking each character's latest index. When a duplicate is found, jump the left pointer past the previous occurrence. The window [left, right] always contains unique characters. Each character is processed once.
def lengthOfLongestSubstring(s):
    char_index = {}  # character -> its latest index
    max_len = 0
    left = 0

    for right in range(len(s)):
        if s[right] in char_index and char_index[s[right]] >= left:
            left = char_index[s[right]] + 1
        char_index[s[right]] = right
        max_len = max(max_len, right - left + 1)

    return max_len
Time: O(n)
Space: O(min(n, charset))
11. Container With Most Water
Medium Two Pointer, Array
▶
📋 Problem Statement
Given n vertical lines at positions (i, 0) to (i, height[i]), find two lines that form the container holding the most water.
🎯 Why This Problem for This Role
Maximizing area under constraints is an optimization problem. The two-pointer approach — start wide and greedily shrink — mirrors capacity optimization: start with maximum resources and intelligently reduce to find the sweet spot.
⚠️ Edge Cases
  • Two lines only → one possible container
  • All same height → widest container wins
  • Strictly increasing heights → optimal involves first line
  • Strictly decreasing heights → optimal involves last line
  • Single very tall line in the middle
🔴 Brute Force
Try every pair of lines. Compute water as min(height) × width. Track maximum.
def maxArea(height):
    max_water = 0

    for i in range(len(height)):
        for j in range(i + 1, len(height)):
            water = min(height[i], height[j]) * (j - i)
            max_water = max(max_water, water)

    return max_water
Time: O(n²)
Space: O(1)
🟢 Optimal Solution
Start with the widest container (left=0, right=n-1). The bottleneck is the shorter line. Moving the taller line inward can only decrease width without helping height, so always move the shorter line. This greedy choice guarantees we don't miss the optimal pair.
def maxArea(height):
    left, right = 0, len(height) - 1
    max_water = 0

    while left < right:
        water = min(height[left], height[right]) * (right - left)
        max_water = max(max_water, water)

        # Move the shorter line inward (it's the bottleneck)
        if height[left] < height[right]:
            left += 1
        else:
            right -= 1

    return max_water
Time: O(n)
Space: O(1)
15. 3Sum
Medium Two Pointer, Sorting, Array
▶
📋 Problem Statement
Given an integer array nums, return all unique triplets [nums[i], nums[j], nums[k]] such that they sum to zero. No duplicate triplets.
🎯 Why This Problem for This Role
Sorting + two-pointer is a pattern that tests algorithmic fluency. Apple expects senior candidates to handle this cleanly, including duplicate skipping. The constraint-satisfaction pattern (find elements meeting a target) appears in resource matching problems.
⚠️ Edge Cases
  • Array shorter than 3 → return []
  • No valid triplets → return []
  • [0,0,0] → [[0,0,0]]
  • Many duplicates requiring careful skip logic
  • All positive or all negative → return []
🔴 Brute Force
Try all triplets. Use a set of tuples to avoid duplicates. Very slow.
def threeSum(nums):
    nums.sort()
    result = set()

    for i in range(len(nums)):
        for j in range(i+1, len(nums)):
            for k in range(j+1, len(nums)):
                if nums[i] + nums[j] + nums[k] == 0:
                    result.add((nums[i], nums[j], nums[k]))

    return [list(t) for t in result]
Time: O(n³)
Space: O(n) for result set
🟢 Optimal Solution
Sort the array. Fix one element, then use two pointers on the remaining range to find pairs that sum to the negative of the fixed element. Skip duplicates at both the outer loop and inner pointers to avoid duplicate triplets.
def threeSum(nums):
    nums.sort()
    result = []

    for i in range(len(nums) - 2):
        # Skip duplicate first elements
        if i > 0 and nums[i] == nums[i-1]:
            continue

        # Two pointer for remaining two elements
        left, right = i + 1, len(nums) - 1

        while left < right:
            total = nums[i] + nums[left] + nums[right]

            if total < 0:
                left += 1
            elif total > 0:
                right -= 1
            else:
                result.append([nums[i], nums[left], nums[right]])
                # Skip duplicates
                while left < right and nums[left] == nums[left+1]:
                    left += 1
                while left < right and nums[right] == nums[right-1]:
                    right -= 1
                left += 1
                right -= 1

    return result
Time: O(n²)
Space: O(1) extra (excluding output)
713. Subarray Product Less Than K
Medium Sliding Window, Array
▶
📋 Problem Statement
Given an array of positive integers nums and an integer k, return the number of contiguous subarrays where the product of all elements is strictly less than k.
🎯 Why This Problem for This Role
Finding windows that satisfy a threshold constraint maps directly to: 'how many time windows have aggregate load below our SLA limit?' This is a core capacity monitoring pattern — counting compliant intervals.
⚠️ Edge Cases
  • k <= 1 → return 0 (all positive, product >= 1)
  • Single element >= k → doesn't count
  • All elements are 1 → every subarray has product 1
  • Product overflows (use careful multiplication)
  • Empty array → return 0
🔴 Brute Force
For each starting index, extend the window and track the running product. Stop when product >= k (since all elements are positive, further extension only increases product).
def numSubarrayProductLessThanK(nums, k):
    if k <= 1:
        return 0

    count = 0
    for i in range(len(nums)):
        product = 1
        for j in range(i, len(nums)):
            product *= nums[j]
            if product < k:
                count += 1
            else:
                break

    return count
Time: O(n²)
Space: O(1)
🟢 Optimal Solution
Sliding window maintaining a running product. When product >= k, shrink from the left. For each right position, the number of valid subarrays ending there is (right - left + 1). Each element enters and leaves the window at most once.
def numSubarrayProductLessThanK(nums, k):
    if k <= 1:
        return 0

    count = 0
    product = 1
    left = 0

    for right in range(len(nums)):
        product *= nums[right]

        while product >= k:
            product //= nums[left]
            left += 1

        # All subarrays ending at right with start in [left, right]
        count += right - left + 1

    return count
Time: O(n)
Space: O(1)

20. Valid Parentheses
Easy Stack, String
▶
📋 Problem Statement
Given a string containing '(', ')', '{', '}', '[', ']', determine if the input string is valid: matching brackets, correct order, every close has a matching open.
🎯 Why This Problem for This Role
Stack-based matching is an Apple phone screen staple. Beyond interviews, bracket/delimiter validation is used in config file parsing, query validation, and infrastructure-as-code linting.
⚠️ Edge Cases
  • Empty string → True
  • Single bracket → False
  • Only opening brackets → False
  • Only closing brackets → False
  • Interleaved types: '([{}])' → True
  • Wrong order: '([)]' → False
🔴 Brute Force
Repeatedly remove adjacent matching pairs until none remain. If the string is empty, it was valid. Conceptually simple but inefficient due to repeated string operations.
def isValid(s):
    # Repeatedly remove innermost valid pairs
    while '()' in s or '[]' in s or '{}' in s:
        s = s.replace('()', '')
        s = s.replace('[]', '')
        s = s.replace('{}', '')

    return s == ''
Time: O(n²) — each replace is O(n), up to n/2 rounds
Space: O(n) for string copies
🟢 Optimal Solution
Use a stack. Push opening brackets. On closing brackets, check if the top of the stack is the matching opener. If not, invalid. At the end, the stack must be empty. One pass, each character handled exactly once.
def isValid(s):
    stack = []
    matching = {')': '(', ']': '[', '}': '{'}

    for char in s:
        if char in matching:
            # Closing bracket: check top of stack
            if not stack or stack[-1] != matching[char]:
                return False
            stack.pop()
        else:
            # Opening bracket: push to stack
            stack.append(char)

    return len(stack) == 0
Time: O(n)
Space: O(n)
49. Group Anagrams
Medium Hash Table, String, Sorting
▶
📋 Problem Statement
Given an array of strings, group the anagrams together. Return the groups in any order.
🎯 Why This Problem for This Role
Hash-based grouping is the pattern behind log categorization, metric aggregation, and error classification. At Apple, you'd group similar resource usage patterns to identify common capacity profiles.
⚠️ Edge Cases
  • Single string → one group with one element
  • Empty string in input → groups with other empty strings
  • All strings are anagrams → one big group
  • No anagrams → each string in its own group
  • Strings of different lengths can't be anagrams
🔴 Brute Force
For each string, compare sorted versions against all others. Very slow with repeated sorting and n² comparisons.
def groupAnagrams(strs):
    groups = []
    used = [False] * len(strs)

    for i in range(len(strs)):
        if used[i]:
            continue
        group = [strs[i]]
        used[i] = True

        for j in range(i + 1, len(strs)):
            if not used[j] and sorted(strs[i]) == sorted(strs[j]):
                group.append(strs[j])
                used[j] = True

        groups.append(group)

    return groups
Time: O(n² × k log k) where k = max string length
Space: O(n × k)
🟢 Optimal Solution
Use sorted string as a hash key — all anagrams sort to the same key. Group into a defaultdict. One pass through the array, each string sorted once. Alternative O(n × k) approach: use character frequency tuple as key instead of sorting.
def groupAnagrams(strs):
    from collections import defaultdict

    groups = defaultdict(list)

    for s in strs:
        # Sort the string as a canonical key
        key = tuple(sorted(s))
        groups[key].append(s)

    return list(groups.values())
Time: O(n × k log k) where k = max string length
Space: O(n × k)
394. Decode String
Medium Stack, String
▶
📋 Problem Statement
Given an encoded string like '3[a2[c]]', decode it. The pattern k[encoded_string] means repeat the encoded_string k times. Example: '3[a2[c]]' → 'accaccacc'.
🎯 Why This Problem for This Role
Stack-based nested structure parsing is relevant to processing configuration templates, infrastructure-as-code expansions, and nested resource definitions. Tests your ability to handle recursive structures iteratively.
⚠️ Edge Cases
  • No encoding: 'abc' → 'abc'
  • Single level: '3[a]' → 'aaa'
  • Nested: '2[a3[b]]' → 'abbbabbb'
  • Multiple sequential: '2[a]3[b]' → 'aabbb'
  • Large repeat count: '100[a]'
🔴 Brute Force
Recursive parsing. When we see a digit, parse the number, skip '[', recurse for inner content, skip ']'. Letters are appended directly. Clean but uses call stack proportional to nesting depth.
def decodeString(s):
    # Recursive approach (conceptually simpler)
    def decode(s, i):
        result = ""

        while i < len(s) and s[i] != ']':
            if s[i].isalpha():
                result += s[i]
                i += 1
            elif s[i].isdigit():
                # Parse the number
                num = 0
                while i < len(s) and s[i].isdigit():
                    num = num * 10 + int(s[i])
                    i += 1
                i += 1  # skip '['
                # Recurse for inner content
                inner, i = decode(s, i)
                i += 1  # skip ']'
                result += num * inner
            else:
                i += 1

        return result, i

    return decode(s, 0)[0]
Time: O(n × max_repeat) where n = output length
Space: O(n) for recursion depth
🟢 Optimal Solution
Iterative stack approach. On '[', push the current string and number, reset both. On ']', pop and combine: previous_string + count × current_string. Letters accumulate in current_str. Digits build current_num. The stack depth equals the nesting level.
def decodeString(s):
    stack = []
    current_str = ""
    current_num = 0

    for char in s:
        if char.isdigit():
            current_num = current_num * 10 + int(char)
        elif char == '[':
            # Save current state and start fresh
            stack.append((current_str, current_num))
            current_str = ""
            current_num = 0
        elif char == ']':
            # Pop and build: previous_str + num * current_str
            prev_str, num = stack.pop()
            current_str = prev_str + num * current_str
        else:
            current_str += char

    return current_str
Time: O(n × max_repeat) where n = output length
Space: O(n) for stack
